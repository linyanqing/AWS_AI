## 🪙 1. Cost Optimization (GENCOST)

### 📐 The Architecture Strategy

The primary threat to an enterprise GenAI budget is unmonitored tracing, uncompressed long-term multi-agent states, and real-time execution of heavy, non-time-sensitive workloads. Under `GENCOST`, we enforce the **W-S-C-I Context Engineering Framework** (*Write* clean prompts, *Select* context vectors, *Compress* summaries, *Isolate* per-agent context) to control token bloat. We minimize unnecessary real-time overhead by adjusting logging sampling rates, implementing Bedrock Prompt Caching for repetitive contexts, and offloading background analysis to asynchronous batch processing.

### 🏢 In the AWS Console

* **CloudWatch Application Signals:** Navigate to **CloudWatch** $\rightarrow$ **Application Signals** $\rightarrow$ **Settings**. Under *GenAI Observability Sampling*, locate your active backend application or Bedrock Agent, click **Edit**, and set the *Trace Indexing %* to **1**.
* **Bedrock Batch Job Setup:** Navigate to **Amazon Bedrock** $\rightarrow$ **Assessment and deployment** $\rightarrow$ **Batch inference**. Click **Create batch inference job**, select your model (`us.anthropic.claude-3-5-sonnet...`), and point the input configuration to your raw payload manifest file inside an encrypted S3 bucket.

### 💻 The Code Implementation: Prompt Caching & Asynchronous Batch Audits

```python
import boto3

bedrock = boto3.client(service_name="bedrock", region_name="ap-southeast-2")
bedrock_runtime = boto3.client(service_name="bedrock-runtime", region_name="ap-southeast-2")

def processing_node_with_cache(state: dict):
    """
    🪙 GENCOST: Leverages explicit prompt cache markers to stop token burn
    on persistent, multi-turn conversational or documentation contexts.
    """
    user_message = state["prompt"]
    large_reference_document = state["large_context_data"]  # e.g., 30,000 tokens

    # Constructing the message block structure with explicit cache control
    messages = [
        {
            "role": "user",
            "content": [
                {
                    "type": "text",
                    "text": large_reference_document,
                    "cache_control": {"type": "ephemeral"}  # Bypasses 80% of processing fees on re-run
                },
                {
                    "type": "text",
                    "text": f"Evaluate the following against our architecture: {user_message}"
                }
            ]
        }
    ]

    response = bedrock_runtime.converse(
        modelId="us.anthropic.claude-3-5-sonnet-20241022-v2:0",
        messages=messages
    )
    return response["output"]["message"]["content"][0]["text"]

def submit_serverless_batch_audit():
    """
    Asynchronous batch processing gives a flat 50% discount vs On-Demand tokens
    """
    response = bedrock.create_model_invocation_job(
        jobName="Nightly_Compliance_Bulk_Audit",
        modelId="us.anthropic.claude-3-5-sonnet-20241022-v2:0",
        roleArn="arn:aws:iam::123456789012:role/BedrockBatchProcessingRole",
        inputDataConfig={"s3InputUri": "s3://rackspace-sydney-vault/batch-inputs/pending_loans.json"},
        outputDataConfig={"s3OutputUri": "s3://rackspace-sydney-vault/batch-outputs/"}
    )
    return response["jobArn"]

```

## 🏎️ 2. Operational Excellence (GENOPS)

### 📐 The Architecture Strategy

Enforcing the Open-Closed Principle (OCP) across enterprise platforms means completely decoupling system prompt optimization from core backend deployment packages. We leverage **Amazon Bedrock Prompt Management** to isolate prompt lifecycles. For runtime observability, we eliminate black-box monitoring by establishing the **AgentCore Observability Engine** integrated with the open-source **OpenTelemetry (OTEL) standard**, allowing granular visualization of internal reasoning paths, tool calls, and orchestration steps.

### 🏢 In the AWS Console

* **Prompt Management Registries:** Navigate to **Amazon Bedrock** $\rightarrow$ **Orchestration** $\rightarrow$ **Prompt management**. Click **Create prompt**, write your system instructions, and declare dynamic parameters using the `{{user_query}}` syntax.
* **Version Aliasing:** Lock down your changes by creating an immutable release version. Go to the **Aliases** tab for that prompt, click **Create alias**, name it `PROD-ACTIVE`, and map it directly to your target version.
* **AgentCore Observability:** Navigate to **Amazon Bedrock** $\rightarrow$ **Orchestration** $\rightarrow$ **Agents**. Select your production agent and scroll to **Telemetry and Logs**. Toggle *CloudWatch Logs* and *OTEL Export* to **Enabled**.

### 💻 The Code Implementation: Decoupled Prompt Hydration via Bedrock Aliases

```python
import boto3

bedrock_control = boto3.client(service_name="bedrock", region_name="ap-southeast-2")
bedrock_runtime = boto3.client(service_name="bedrock-runtime", region_name="ap-southeast-2")

def execute_decoupled_prompt(customer_query: str):
    # Core software is CLOSED to modification. It targets an immutable prompt alias ARN.
    PROMPT_ALIAS_ARN = "arn:aws:bedrock:ap-southeast-2:123456789012:prompt/LOAN_ROUTER/aliases/PROD-ACTIVE"

    # 1. Fetch system instructions dynamically from the cloud configuration registry
    prompt_config = bedrock_control.get_prompt(promptIdentifier=PROMPT_ALIAS_ARN)
    system_template = prompt_config['variants'][0]['templateText']

    # 2. Hydrate configuration template variables smoothly
    final_instructions = system_template.replace("{{user_query}}", customer_query)

    # 3. Route clean payload to the inference engine with OpenTelemetry tracing contexts attached
    response = bedrock_runtime.converse(
        modelId="us.anthropic.claude-3-5-sonnet-20241022-v2:0",
        messages=[{"role": "user", "content": [{"text": final_instructions}]}]
    )
    return response['output']['message']['content'][0]['text']

```

## 🛡️ 3. Reliability (GENREL)

### 📐 The Architecture Strategy

Enterprise reliability requires isolating agent blast radiuses and handling non-deterministic failures through structured orchestration frameworks. We utilize **Strands Agents** to build predictable multi-agent communication networks. We separate long-running agent tasks from real-time endpoints using the **SNS + SQS Fan-Out pattern**. To protect against region-wide model failures or latency saturation, we implement programmatic circuit breakers and automated rollback mechanisms that dynamically flip traffic between dedicated Provisioned Throughput endpoints and elastic Serverless pools.

### 🏢 In the AWS Console

* **Fan-Out Messaging Architecture:** Navigate to **Amazon SNS** $\rightarrow$ **Topics** $\rightarrow$ **Create topic**. Select *Standard*, and name it `AgentTransactionStream`. Next, navigate to **Amazon SQS** $\rightarrow$ **Create queue** to stand up decoupled sub-agent processing tracks (`fraud-queue`, `compliance-queue`). Return to your SNS topic, click **Create subscription**, select *Amazon SQS*, and bind the queues.
* **AWS Step Functions Stage Gates:** Navigate to **Step Functions** $\rightarrow$ **State machines** $\rightarrow$ **Create state machine**. Use the Workflow Studio to visually structure your `Task` states, inserting explicit `Catch` parameters to capture Bedrock API faults natively.

### 💻 The Code Implementation: Fan-Out Broadcast with Circuit-Breaking Failover

```python
import boto3
import json
from botocore.exceptions import ClientError

sns = boto3.client(service_name="sns", region_name="ap-southeast-2")
bedrock_runtime = boto3.client(service_name="bedrock-runtime", region_name="ap-southeast-2")

TOPIC_ARN = "arn:aws:sns:ap-southeast-2:123456789012:AgentTransactionStream"
DEDICATED_PT_ARN = "arn:aws:bedrock:ap-southeast-2:123456789012:provisioned-model/sydney-prod-fast-lane"
BACKUP_MODEL_ID = "us.anthropic.claude-3-5-sonnet-20241022-v2:0"

def broadcast_transaction_event(account_id: str, payload_summary: str):
    """Isolates sub-agent blast radiuses via asynchronous fanning out"""
    event_message = {"account_id": account_id, "summary": payload_summary}
    sns.publish(TopicArn=TOPIC_ARN, Message=json.dumps(event_message))

def reliable_inference_fallback(user_prompt: str):
    """Executes call to Provisioned Capacity; trips circuit breaker natively to Serverless pool on failure"""
    try:
        # Attempt execution on dedicated hardware lane
        response = bedrock_runtime.converse(
            modelId=DEDICATED_PT_ARN,
            messages=[{"role": "user", "content": [{"text": user_prompt}]}]
        )
        return response['output']['message']['content'][0]['text']
    except ClientError as e:
        # Capture Throttling or Noisy-Neighbor infrastructure failures
        if e.response['Error']['Code'] in ['ThrottlingException', 'ServiceUnavailableException']:
            print("🚨 Circuit Breaker Tripped! Falling back to multi-tenant serverless pool.")
            # Trigger automated rollback detour path
            backup_response = bedrock_runtime.converse(
                modelId=BACKUP_MODEL_ID,
                messages=[{"role": "user", "content": [{"text": user_prompt}]}]
            )
            return backup_response['output']['message']['content'][0]['text']
        raise e

```

## ⚡ 4. Performance Efficiency (GENPERF)

### 📐 The Architecture Strategy

Real-time customer-facing agent architectures cannot tolerate the latency profiles of traditional request-response polling loops. To guarantee tight end-to-end SLAs, we deploy our workloads directly onto the **AgentCore Harness**. We eliminate external database lookup latency by leveraging **AgentCore Memory (Managed Short/Long Term)**. This allows the system to natively persist user-namespaced data inside the cloud runtime boundary, avoiding slow vector index fetches for core preferences. Finally, we connect the stack to an **API Gateway WebSocket** layer combined with `converse_stream` for sub-second, multi-turn token streaming.

### 🏢 In the AWS Console

* **AgentCore Harness Initialization:** Navigate to **Amazon Bedrock** $\rightarrow$ **Orchestration** $\rightarrow$ **Agents**. Click *Create Agent*, select the **AgentCore Harness** runtime cluster, and assign your model configuration.
* **Managed Memory Settings:** Inside your active Agent overview panel, find the **Memory Strategy** block. Click **Edit**, toggle *Long-term Memory* to **Enabled**, and specify an auto-expiration window (e.g., 30 days) to keep context windows highly optimized.

### 💻 The Code Implementation: WebSocket Token Streaming via AgentCore Harness

```python
import boto3
import json

def lambda_handler(event, context):
    connection_id = event['requestContext']['connectionId']
    domain_name = event['requestContext']['domainName']
    stage = event['requestContext']['stage']
    user_prompt = json.loads(event['body'])['prompt']
    session_id = json.loads(event['body'])['session_id']

    # Native runtime client optimized for managed AgentCore Harness operations
    agent_runtime = boto3.client(service_name="bedrock-agent-runtime", region_name="ap-southeast-2")
    gateway_api = boto3.client(service_name="apigatewaymanagementapi", endpoint_url=f"https://{domain_name}/{stage}")

    # Invite the agent with tracing and managed memory active
    stream_response = agent_runtime.invoke_agent(
        agentId="SYDNEY-PROD-AGENT-ID",
        agentAliasId="PROD-ACTIVE",
        sessionId=session_id, # AgentCore automatically re-injects short/long term user preferences here
        inputText=user_prompt,
        enableTrace=False
    )

    for event in stream_response["completion"]:
        if "chunk" in event:
            live_token = event["chunk"]["bytes"].decode("utf-8")
            # Push live token straight to user browser interface
            gateway_api.post_to_connection(ConnectionId=connection_id, Data=json.dumps({"token": live_token}))

```

## 🔒 5. Security (GENSEC)

### 📐 The Architecture Strategy

Enterprise-grade agent deployments require absolute data boundaries and protection against prompt injection vector patterns. To achieve strict runtime tenant isolation, the **AgentCore Runtime** isolates user execution environments using ephemeral, secure **microVM sessions**. Each user session runs inside an isolated compute pocket that is completely destroyed immediately upon completion. We layer this with **In-Pipeline PII Masking** and multi-tiered **Amazon Bedrock Guardrails** to scan both user entry vectors and data output streams synchronously.

### 🏢 In the AWS Console

* **Session microVM Controls:** Handled implicitly when using the *Amazon Bedrock AgentCore Runtime* API layer.
* **Guardrail Gateways:** Navigate to **Amazon Bedrock** $\rightarrow$ **Safeguards** $\rightarrow$ **Guardrails**. Click **Create guardrail**, and set your regional context strictly to *Asia Pacific (Sydney)* to protect regulatory data sovereignty.
* **Filters & Masking Configurations:** In the *Content Filters* tab, toggle *Prompt attacks* to **High**. In the *Sensitive Information* tab, input entities such as *Email*, *Credit Card*, and *IP Address*. Set their action profiles to **Block Input** and **Mask Output**.

### 💻 The Code Implementation: Isolated Session Execution with Synchronous Perimeter Guardrails

```python
import boto3
from botocore.exceptions import ClientError

agent_runtime = boto3.client(service_name="bedrock-agent-runtime", region_name="ap-southeast-2")

def secure_agent_execution(user_input: str, user_session_id: str):
    """
    🔒 GENSEC: Executes within an isolated, ephemeral microVM environment
    wrapped with dual-sided input/output PII masking.
    """
    try:
        response = agent_runtime.invoke_agent(
            agentId="SYDNEY-COMPLIANCE-AGENT",
            agentAliasId="PROD-ACTIVE",
            sessionId=user_session_id, # Enforces strict microVM containment boundary
            inputText=user_input
            # Guardrails are bound directly to the agent's definition inside the control plane
        )

        final_output = ""
        for event in response["completion"]:
            if "chunk" in event:
                final_output += event["chunk"]["bytes"].decode("utf-8")
            elif "trace" in event:
                # Intercept trace logs if a guardrail intervention trips at runtime
                trace = event["trace"]["trace"]
                if "guardrailInterception" in trace:
                    print("🚨 Security Alert: Malicious prompt attempt or PII leak neutralized!")

        return final_output

    except ClientError as e:
        print(f"Security Policy Fault: {e}")
        raise e

```

## 🍃 6. Sustainability (GERSUST)

### 📐 The Architecture Strategy

Sustainability in modern GenAI means minimizing the carbon footprint and computational intensity of every single transaction. We move completely away from always-on monolithic server instances by exploiting the multi-tenant **Amazon Bedrock Managed Ecosystem**. We use intentional **Context Selection** to right-size our prompt structures. Finally, we implement **Dynamic Intent Routing**—using a low-power model like Amazon Nova Micro for structural evaluations and offloading requests to heavy frontier engines only when highly complex reasoning is explicitly demanded.

### 🏢 In the AWS Console

* **The AWS Customer Carbon Footprint Tool:** Open the **Billing and Cost Management Console** $\rightarrow$ look down the left-hand navigation column $\rightarrow$ select **Carbon Footprint Tool**. Use this interface to actively monitor the carbon emissions metrics generated by your architecture.
* **Model Catalog Selection:** Navigate to **Amazon Bedrock** $\rightarrow$ **Model catalog**. Right-size your workflows by utilizing lightweight, high-performance-per-watt models (such as `amazon.nova-micro-v1:0`) for non-complex, transactional workflows.

### 💻 The Code Implementation: Carbon-Minimized Intent Routing

```python
import boto3

bedrock_runtime = boto3.client(service_name="bedrock-runtime", region_name="ap-southeast-2")

def sustainable_intent_router(user_query: str):
    """
    🍃 GERSUST: Routes simple queries to ultra-low-power compute running on
    highly efficient AWS custom Silicon, avoiding carbon-heavy frontier models when unnecessary.
    """
    SUSTAINABLE_MODEL_ID = "amazon.nova-micro-v1:0"

    routing_prompt = f"Classify the user intent into SIMPLE or COMPLEX. Return only the token. Query: {user_query}"

    response = bedrock_runtime.converse(
        modelId=SUSTAINABLE_MODEL_ID,
        messages=[{"role": "user", "content": [{"text": routing_prompt}]}]
    )
    intent = response["output"]["message"]["content"][0]["text"].strip()

    if "SIMPLE" in intent:
        print("🍃 Sustainability Match: Handled via low-emission compute lane.")
        return f"Processed efficiently via {SUSTAINABLE_MODEL_ID}"
    else:
        print("🚀 Scaling up to deeper reasoning frontier architecture.")
        complex_response = bedrock_runtime.converse(
            modelId="us.anthropic.claude-3-5-sonnet-20241022-v2:0",
            messages=[{"role": "user", "content": [{"text": user_query}]}]
        )
        return complex_response["output"]["message"]["content"][0]["text"]

```

## 🧪 7. NEW ADDITION: Evaluation & Trust (GENEVAL)

### 📐 The Architecture Strategy

As highlighted in the AWS Summit briefing, the main roadblock to moving agents out of demo status is non-deterministic output drift. Under `GENEVAL`, we establish a **5-Step Continuous Evaluation Loop**: *Ground Truth Datasets $\rightarrow$ Offline Evaluation $\rightarrow$ Safe Deployment $\rightarrow$ Online Monitoring $\rightarrow$ Continuous Improvement*. We utilize **AgentCore Evaluations** to run real-time quality intelligence scoring and custom business metrics against production transactions, feeding edge cases back into our test beds.

### 🏢 In the AWS Console

* **Model Evaluation Configuration:** Navigate to **Amazon Bedrock** $\rightarrow$ **Assessment and deployment** $\rightarrow$ **Model evaluation**. Click **Create evaluation job** and select **Automated**.
* **Selecting Ground Truth Datasets:** Upload your master gold-standard dataset (JSONL format) from S3.
* **RAG Retrieval Quality Metrics:** Select your evaluation targets:
* **Faithfulness (Groundedness):** Measures if the model output *only* uses information from the retrieved context (flagging hallucinations).
* **Answer Relevance:** Checks if the generated response directly answers the user's prompt.
* **Context Relevance:** Validates the precision and noise levels of your vector index search results.



### 💻 The Code Implementation: Parsing AgentCore Traces for Real-Time Runtime Evaluation

```python
import boto3

agent_runtime = boto3.client(service_name="bedrock-agent-runtime", region_name="ap-southeast-2")

def invoke_and_evaluate_runtime_agent(user_query: str, session_id: str):
    """
    🧪 GENEVAL: Invokes agent with active tracing to unpack and evaluate
    the internal RAG context and model rationale stream in real time.
    """
    response = agent_runtime.invoke_agent(
        agentId="SYDNEY-PROD-AGENT",
        agentAliasId="PROD-ACTIVE",
        sessionId=session_id,
        inputText=user_query,
        enableTrace=True  # 👈 CRITICAL: Instructs AgentCore to append the internal reasoning path
    )

    final_answer = ""
    for event in response["completion"]:
        if "chunk" in event:
            final_answer += event["chunk"]["bytes"].decode("utf-8")

        elif "trace" in event:
            trace_data = event["trace"]["trace"]

            # Runtime Evaluation Layer 1: Verify the Context Relevance of the RAG Pull
            if "knowledgeBaseLookup" in trace_data:
                lookup = trace_data["knowledgeBaseLookup"]
                print("\n🔍 [RUNTIME RAG EVALUATION] Sources Pulled from Vector Index:")
                for ref in lookup.get("retrievedReferences", []):
                    print(f"   -> Location: {ref['location']['s3Location']['s3LocationUri']}")
                    print(f"   -> Snippet Text: {ref['content']['text'][:80]}...")

            # Runtime Evaluation Layer 2: Audit the Rationale (The model's internal thought loop)
            elif "orchestration" in trace_data:
                orch = trace_data["orchestration"]
                if "rationale" in orch:
                    print(f"🧠 [AGENT CORE RATIONALE]: {orch['rationale']['text']}")
                if "invocationInput" in orch:
                    # Verify if the agent selected the correct Action Group tool schema
                    tool = orch["invocationInput"].get("actionGroupInvocationInput", {})
                    print(f"🛠️ [TOOL CALL AUDIT]: Executed '{tool.get('actionGroupName')}' -> '{tool.get('function')}'")

    return final_answer

```
